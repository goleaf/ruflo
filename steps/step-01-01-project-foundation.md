# STEP 01 — Full project analysis and professional Laravel Todo foundation preparation

## Global rules

Before this step, read:

- `../GLOBAL_RULES.md`
- repository progress files
- previous step results

## Step prompt

Analyze the full project first. Prepare a clean Laravel 13 + Livewire + Flux foundation.

Focus on architecture, routing, authentication state, current frontend stack, translation system, tests, documentation, UI foundation, component reuse, request/validation structure, policies, factories, seeders, and project conventions.

Do not build all features yet. Prepare the base so every later step can be added cleanly.

Required output:
- clean architecture plan
- Livewire/Flux UI foundation
- no Volt architecture
- private-by-default mindset
- translation-ready UI
- test-ready structure
- docs/changelog/progress update
- commit


## Mandatory step checklist

- Inspect current project state.
- Respect Laravel 13 + Livewire + Flux + Tailwind 4.
- Do not use Volt.
- Respect restricted hosting mode.
- Use web UI instead of cron/jobs/artisan.
- Add/update factories and seeders where relevant.
- Add/update request classes and custom validation rules where relevant.
- Translate all visible text and validation messages.
- Add/update tests.
- Update documentation.
- Update changelog.
- Update progress files.
- Review git diff.
- Commit with a meaningful message.

## Final step response format

At the end of this step, report:

- what was implemented
- what was changed
- what tests/checks were run
- what documentation was updated
- what remains unfinished
- what risks remain
- next recommended step
